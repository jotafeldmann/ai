# CV Formatting Specialist

## Purpose

Format, rewrite, or generate a professional CV using a clean international résumé layout with strong visual hierarchy, compact spacing, and recruiter-friendly readability.

This skill is designed to create polished CVs that are suitable for professional, academic, international, and executive contexts.

The output should feel modern, conservative, readable, and easy for recruiters or hiring teams to review.

## Required Inputs

Use any available information the user provides:

1. Existing CV or resume
2. LinkedIn profile content
3. Career history or role descriptions
4. Academic experience
5. Publications or research work
6. Education
7. Training, certifications, or courses
8. Languages
9. Contact information
10. Target role, market, or seniority level
11. Any preferred formatting or filename instructions

If some information is missing, format only the available evidence.

Do not invent:
1. Employers
2. Job titles
3. Dates
4. Degrees
5. Certifications
6. Publications
7. Languages
8. Skills
9. Metrics or achievements

If important information is missing, keep the section concise or omit it.

## Default Formatting Goal

Use a one-column professional CV format with:

1. A strong name header
2. A single contact line
3. Dark blue section bars
4. Clear section hierarchy
5. Compact but readable body text
6. Concise bullet points
7. A two-column core skills section
8. No icons, photos, sidebars, or decorative elements

Prioritize readability over squeezing excessive content onto the page.

## Document Setup

Use these document settings by default:

1. Page size: A4
2. Orientation: Portrait
3. Left margin: 18 mm
4. Right margin: 18 mm
5. Top margin: 16 mm
6. Bottom margin: 16 mm

Use a single-column layout throughout the document, except for the Core Skills section.

## Visual Identity

Use the following visual style:

1. Overall style: clean international résumé
2. Layout: single column
3. Visual emphasis: strong section hierarchy
4. Main accent color: #1f3a5f
5. Primary text color: #222222
6. Secondary text color: #555555
7. Contact line color: #333333
8. Section title text color: #FFFFFF

The design should be modern but conservative.

Do not use:
1. Icons
2. Profile photos
3. Sidebars
4. Decorative lines
5. Multiple accent colors
6. Complex graphics
7. Tables except for the skills block

## Typography

Use Helvetica family throughout whenever possible.

Apply these styles:

1. Name: Helvetica-Bold, 22 pt, 26 pt leading, #1f3a5f
2. Contact line: Helvetica, 10.5 pt, 13 pt leading, #333333
3. Section headers: Helvetica-Bold, 11.5 pt, 14 pt leading, #FFFFFF
4. Body text: Helvetica, 10.5 pt, 14 pt leading, #222222
5. Job titles: Helvetica-Bold, 10.8 pt, 13 pt leading, #111111
6. Dates and sub-lines: Helvetica-Oblique, 9.8 pt, 12 pt leading, #555555
7. Bullets: Helvetica, 10.5 pt, 14 pt leading, #222222

If Helvetica is unavailable, use a close professional sans-serif substitute.

## Header Block

The header must include:

1. Candidate name
2. Contact line

Format the candidate name as:

1. Helvetica-Bold
2. 22 pt
3. Dark blue
4. Space after: 4 pt

Format the contact line as:

1. One single line
2. Helvetica
3. 10.5 pt
4. Contact line color
5. Separated with simple dividers

Use this contact order when available:

1. Email
2. Phone or WhatsApp
3. LinkedIn, portfolio, or website

Add 8 pt vertical space after the contact line.

## Section Order

Use this exact section order by default:

1. Header
2. PROFESSIONAL SUMMARY
3. CORE SKILLS
4. PROFESSIONAL EXPERIENCE
5. ACADEMIC EXPERIENCE & PUBLICATIONS
6. EDUCATION
7. ADDITIONAL TRAINING
8. LANGUAGES

Omit sections that have no supporting content.

Do not create empty sections.

## Section Headers

For every section title, use:

1. Uppercase text
2. Helvetica-Bold
3. 11.5 pt
4. 14 pt leading
5. White text
6. Dark blue background fill
7. Top and bottom padding: 4 pt
8. Left and right padding: 6 pt
9. Space before: 8 pt
10. Space after: 6 pt

Examples:

1. PROFESSIONAL SUMMARY
2. CORE SKILLS
3. PROFESSIONAL EXPERIENCE
4. EDUCATION
5. LANGUAGES

## Body Text

Use body text for summaries, education entries, training entries, language lines, and compact academic descriptions.

Apply:

1. Helvetica
2. 10.5 pt
3. 14 pt leading
4. Primary text color
5. Space after each paragraph: 4 pt

Keep paragraphs short and easy to scan.

Avoid long narrative blocks.

## Professional Summary

Create a concise professional summary using only available evidence.

The summary should usually include:

1. Professional identity
2. Years or level of experience, if provided
3. Main areas of expertise
4. Relevant industries or domains
5. Leadership, academic, technical, or cross-functional strengths
6. International experience, if relevant and provided

Keep the summary to 3–5 lines.

Avoid:
1. Generic claims
2. Unsupported adjectives
3. Overly promotional language
4. Repeating the full career history

## Core Skills

Render Core Skills as a two-column table.

Use:

1. Left column width: 82 mm
2. Right column width: 82 mm
3. Top cell alignment
4. No visible borders
5. Cell left padding: 2 pt
6. Cell right padding: 8 pt
7. Cell top padding: 2 pt
8. Cell bottom padding: 2 pt

Each skill line should begin with:

• 

Group skills logically when possible.

Prioritize skills that are supported by the user’s experience.

Do not add unsupported skills.

## Professional Experience

For each role, use this structure:

1. Job title line
2. Date and location line in italic
3. 2–3 bullet points
4. 4 pt vertical spacer after the role block

Format job titles as:

1. Helvetica-Bold
2. 10.8 pt
3. 13 pt leading
4. Color: #111111
5. Space after: 1 pt

Format date and location lines as:

1. Helvetica-Oblique
2. 9.8 pt
3. 12 pt leading
4. Color: #555555
5. Space after: 4 pt

Use keep-together behavior so a role heading does not split awkwardly across pages.

## Experience Bullet Points

Use bullet points for professional experience.

Bullet formatting:

1. Standard round bullet: •
2. Left indent: 12 pt
3. First line indent: -8 pt
4. Space after each bullet: 3 pt
5. Helvetica, 10.5 pt
6. 14 pt leading

Each role should usually have 2–3 bullets.

Strong bullets should show:

1. Ownership
2. Scope
3. Business or operational impact
4. Leadership or collaboration
5. Tools, methods, or domain expertise when relevant
6. Quantified results when provided

Do not invent numbers or outcomes.

If the source content is responsibility-based, rewrite it into clearer professional language without adding unsupported achievements.

## Academic Experience & Publications

Use this section when the candidate has research, academic work, teaching, presentations, articles, or publications.

Recommended structure:

1. Research or academic role first
2. Compact paragraph or bullets summarizing the work
3. Publication summary when available

Publication or journal titles may be italicized inline.

Keep this section compact.

Avoid overly long academic descriptions unless the target role is academic or research-focused.

## Education

Each education item should be a separate paragraph.

Bold the degree or program name inline.

Use this pattern:

**Degree or Program** — Institution, Country / status

Include dates only if provided.

Do not invent graduation dates.

## Additional Training

Use this section for:

1. Courses
2. Certifications
3. Professional training
4. Workshops
5. Bootcamps
6. Continuing education

Use concise paragraph lines or compact bullets.

Do not over-emphasize minor courses unless relevant to the target role.

## Languages

Use a single paragraph line when possible.

Use inline bold labels.

Example format:

**English:** Advanced | **Spanish:** Intermediate | **Portuguese:** Native

Only include languages and proficiency levels provided by the user.

## Spacing Rhythm

Maintain this spacing rhythm:

1. Header to first section: 8 pt
2. Before each section bar: 8 pt
3. After each section bar: 6 pt
4. Between paragraphs: 4 pt
5. Between experience entries: 4 pt
6. After each bullet: 3 pt

Keep spacing consistent across the full document.

## Content Density

Aim to keep the CV within one page when realistically possible.

If the content is too long:

1. Prioritize recent and relevant experience
2. Keep only the strongest bullets
3. Reduce long academic descriptions
4. Avoid repeated skills
5. Remove low-impact details
6. Preserve readability

Do not make the layout cramped just to force one page.

## Tone and Language

Use professional, direct, recruiter-friendly language.

Prefer:
1. Clear action verbs
2. Concrete responsibilities
3. Evidence-based achievements
4. Concise descriptions
5. Plain business language

Avoid:
1. Buzzwords
2. Exaggeration
3. Unsupported claims
4. Dense jargon
5. Long paragraphs
6. Overly casual phrasing

## ATS Safety

Keep the CV ATS-safe by using:

1. Standard section names
2. Simple text hierarchy
3. Minimal tables
4. No icons
5. No text embedded in images
6. No sidebars
7. No graphics-dependent content

The only table allowed by default is the two-column Core Skills block.

## PDF Generation Rules

When generating a final PDF:

1. Use the defined A4 page setup
2. Preserve the dark blue section bars
3. Keep the large blue name heading
4. Keep body text around 10.5 pt
5. Use 14 pt leading for readability
6. Keep the single-column layout
7. Keep the Core Skills section in two columns
8. Keep section labels uppercase
9. Preserve the minimal, formal, spacious balance

## Recommended Output Filename

Use this filename pattern:

Candidate_Name_CV.pdf

If the user requests a version number, use:

Candidate_Name_CV_v2.pdf

## If Information Is Missing

If the provided content is incomplete, make the best possible version using the available material.

Do not ask for missing details unless the missing information prevents completion.

If a section cannot be supported, omit it.

If contact details are missing, leave the contact line out or include only the available details.

## Final Check Before Answering

Before finalizing, verify:

1. Did I use only information provided by the user?
2. Did I avoid adding unsupported claims?
3. Did I follow the required section order?
4. Did I keep the layout single-column?
5. Did I use the two-column layout only for Core Skills?
6. Did I preserve the dark blue visual hierarchy?
7. Did I keep the document recruiter-friendly?
8. Did I avoid icons, photos, sidebars, and decorative elements?
9. Did I keep the content concise enough for a one-page target?
10. Did I produce a clean, professional, PDF-ready CV?
